## Reporting a Vulnerability

We take security seriously. If you believe you have found a security vulnerability in Audacity Setup Guide, please report it to us as described below.

### How to Report

Please do **NOT** report security vulnerabilities through public GitHub issues. Instead, please send an email to security@audacityaudio.org with the following information:

- Type of issue (e.g. buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit the issue

### Response Timeline

- We will acknowledge receipt of your report within 48 hours
- We will provide a detailed response within 7 days indicating our next steps
- We will keep you informed of our progress throughout the resolution process
- We may ask for additional information or guidance

### Security Updates

Security updates will be released as soon as possible after a vulnerability is confirmed and a fix is developed. We will:

1. Issue a new release with the security fix
2. Update this security policy if needed
3. Publish a security advisory on GitHub
4. Credit you for the discovery (if desired)

## Security Best Practices

When using this setup guide:

- Always download Audacity from official sources
- Verify checksums of downloaded files
- Run the setup with appropriate privileges only
- Keep your system updated with latest security patches
- Use antivirus software when working with downloaded files

## Scope

This security policy applies to:

- The setup guide and tools provided in this repository
- Associated documentation and scripts
- Installation and configuration processes

This policy does not cover:

- The Audacity application itself (report to the main Audacity project)
- Third-party plugins or extensions
- User-generated content or configurations
